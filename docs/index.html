Go-Boiler ?

Go-boiler is crafted by combining existing golang libraries. It's not a framework. It's a boliler-plate to give you a head start on your project. Its gives you basic folder structure and essentail functions required in website/ webapp.

According to golang philosophy you don't need framework, instead of using framework they says integrate libraries according to your need. This boiler-plate follow the same philosophy and so it has just provided routing and response rendering. Everything else is left on developer ( Yes! that's you ) including database.

Have tried to intergrate minimum libraries so no have anything that's unused from this boiler-plate.

Features

Go-boiler is built-on top of existing libraries. It gives very basic facilites need for website/ webapps those are listed below.

  • Web Server
  • Routing
    • Static Routes
    • Dynamic Routes
  • Response Rendering
    • HTML Rendering
    • Text Rendering
    • JSON Response

Installation


You need source code of go-boiler in your machine. You can either clone the project from github or download source code.

For clonning project
$ git clone https://www.github.com/mchampaneri/go-boiler.git

Folder Structure

Getting Started

You have to build and start application server for your application to start your application.

    Build server
  • Open Terminal / CMD
  • Goto root folder of your project
    $ application\ :
  • Travers to server-logic folder
    $ application\server-logic\ :
  • Generate server in parent fodler
    $ appliaction\server-logic\ : go build -o ../xerver
    Start server
  • Open Terminal / CMD
  • Goto root folder of your project
    $ application\ :
  • Start xerver executable
    $ appliactions\: ./xerver

After starting server open http://localhost:8081 in browser. You should see welcome page.

Welcome page you are seeing is actully set with static routing.

Static routes are defined in static.json 

                         {
                            "pages":[
                              {
                                "url":"/",
                                "view":"index.html"
                              }
                            ]
                          }
