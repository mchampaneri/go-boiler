Go-Boiler ?

Go-boiler is crafted by combining existing golang libraries. It's not a framework. It's a boliler-plate to give you a head start on your project. Its gives you basic folder structure and essentail functions required in website/ webapp.

According to golang philosophy you don't need framework, instead of using framework they says integrate libraries according to your need. This boiler-plate follow the same philosophy and so it has just provided routing and response rendering. Everything else is left on developer ( Yes! that's you ) including database.

Features

Go-boiler is built-on top of existing libraries. It gives very basic facilites need for website/ webapps those are listed below.

  • Web Server (Golang net/http)
  • Routing (Gorilla Mux)
    • Static Routes
    • Dynamic Routes
  • Response Rendering (Cloudykit/jet & enocde/json)
    • HTML Rendering
    • Text Rendering
    • JSON Response
  • CSRF (Gorilla/csrf)

Getting Started

You have to build and start application server for your application to start your application.

    Build server
  • Open Terminal / CMD
  • Goto root folder of your project
    $ application\ :
  • Travers to server-logic folder
    $ application\server-logic\ :
  • Generate server in parent folder
    $ appliaction\server-logic\ : go build -o ../xerver
    Start server
  • Open Terminal / CMD
  • Goto root folder of your project
    $ application\ :
  • Start xerver executable
    $ appliactions\: ./xerver

Open URL http://localhost:8081.

Static Routing

When you open URL localhost:8081. You get welcome page. It's actully comming from static routes mapping. 

                           
                            {
                              "pages":[{
                                      "url":"/",
                                      "view":"index.html"
                                     },
                                     {
                                      "url":"/about",
                                      "view":"site/about.html"
                                    }]
                            }

static.json

http://yoursite-name.domain/about.html

Path to the view file ( Relative path to view folder ) view

  • index.html
  • site
    • about.html

All static routes has to be defined inside static.json as pair of URL and page.

For example we have index.html page and about.html page with "/" and "/about" route. For those route static.json and folder arrangment of files are shown in example.


Static routes has to be defined in static.json. Application need to be restarted to get static routes changes in effect. Static routes does not process any render time data. So it should be use only for complete static data.( Thouhg, you may fetch dynamic data using ajax )
